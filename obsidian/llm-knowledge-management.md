# LLM으로 지식 유지하는 법

## 문제: LLM은 기억이 없다

대화가 끝나면 LLM은 이전 내용을 잊는다.  
→ 매번 처음부터 설명해야 하는 비효율 발생.

## 해결: 외부 지식 저장소 (이 레포)

```
대화 시작 →  관련 노트 제공 → LLM이 컨텍스트 파악 → 작업
                ↑
          study-repo (지식 저장소)
```

---

## 실천 방법

### 1. 배운 것을 즉시 노트화
```
Claude Code와 작업 후:
- 새로 알게 된 패턴 → 해당 노트에 추가
- 삽질한 내용 → "주의사항" 섹션에 추가
- 좋은 프롬프트 패턴 → codebase-analysis.md에 추가
```

### 2. 다음 대화 시작 시 노트 제공
```
"이 프로젝트는 이런 구조야: [README 내용]
 지난번에 이런 걸 배웠어: [관련 노트 내용]
 오늘 하고 싶은 것: ..."
```

### 3. 노트를 LLM에게 직접 학습시키기
```bash
# Claude Code에 전체 study-repo 컨텍스트 제공
cat llm/*.md | pbcopy  # 클립보드에 복사 후 붙여넣기

# 또는 Claude Code가 직접 읽게 하기
# "study-repo/llm 폴더의 내용을 참고해서 작업해줘"
```

---

## Obsidian + Git으로 지식 누적하기

```
학습/작업
   ↓
노트 작성/수정 (Obsidian)
   ↓
git commit & push
   ↓
GitHub에 누적 (포트폴리오 겸용)
   ↓
다음 작업 시 관련 노트 LLM에 제공
```

---

## 카파시 정신과 연결

단순히 LLM이 생성한 내용을 저장하는 게 아니라,  
**내가 이해한 내용을 내 언어로 정리**하는 것이 핵심.

> LLM이 설명해줬다 → 이해했다 → 내 말로 다시 써봤다 → 진짜 아는 것
