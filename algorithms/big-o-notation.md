# Big-O 표기 — 정석부터

> Phase 1의 첫 문서. 읽기만으로는 안 된다 — 맨 아래 **직접 풀어보기**를 손으로 풀고,
> 막히면 다시 위로 올라온다. (깊이는 문제를 직접 풀 때 생긴다.)

## 1. 왜 필요한가 — "빠르다"를 객관화

"이 코드 빠르다"는 컴퓨터·언어·입력에 따라 다르다. 그래서 **입력 크기 n이 커질 때
연산 횟수가 어떤 비율로 늘어나는가**라는, 기계와 무관한 척도가 필요하다. 그게 Big-O다.

핵심: Big-O는 **실행 시간(초)이 아니라 증가율(growth rate)** 이다.

## 2. 정의 (점근 표기)

함수 T(n)이 충분히 큰 n에서 어떤 상수배의 g(n)을 넘지 않으면 `T(n) = O(g(n))`.

```
T(n) = O(g(n))  ⇔  어떤 상수 c>0, n₀가 있어서  n ≥ n₀ 이면  T(n) ≤ c·g(n)
```
- **O (빅오)**: 상한 (최악의 경우, "이보다 나쁘진 않다")
- **Ω (오메가)**: 하한 (최선)
- **Θ (세타)**: 상·하한이 같을 때 (정확한 증가율)

대화에서 "O(n)"이라 하면 보통 **Θ(n) 수준의 최악 상한**을 뜻한다.

## 3. 두 가지 단순화 규칙 (정석)

### 규칙 ①: 상수는 버린다
`3n`, `100n`, `0.5n` 모두 **O(n)**. 이유: n이 커지면 상수배는 증가율을 안 바꾼다.
→ Big-O는 "기계·언어 차이(상수배)"를 일부러 무시해 **본질적 증가율**만 본다.

### 규칙 ②: 지배항만 남긴다
`n² + n + 100` → **O(n²)**. n이 커지면 n²가 압도하므로 작은 항은 무시.
```
n=1000일 때:  n² = 1,000,000  /  n = 1,000  /  상수 = 100
→ n²가 99.9% 지배
```

## 4. 자주 나오는 복잡도 (작은 → 큰)

| 표기 | 이름 | 대표 예 | n=1,000,000이면 |
|------|------|---------|----------------|
| O(1) | 상수 | 배열 인덱싱, 해시 조회 | 1번 |
| O(log n) | 로그 | 이진탐색 | 약 20번 |
| O(n) | 선형 | 배열 한 번 순회 | 100만 번 |
| O(n log n) | 선형로그 | 병합·퀵 정렬 | 약 2천만 번 |
| O(n²) | 제곱 | 이중 루프 | 1조 번 (사실상 불가) |
| O(2ⁿ) | 지수 | 부분집합 완전탐색 | 우주가 끝나도 못 끝냄 |

> 직관: **O(log n)은 "입력이 2배여도 +1번"**, O(n²)는 "입력 2배면 4배"가 된다.

## 5. 분석하는 법 (단계별)

```
1. 기본 연산 1회 = O(1)
2. 순차 실행 = 더한 뒤 지배항 (O(a)+O(b) → 큰 쪽)
3. 루프 = 반복 횟수 × 본문
4. 중첩 루프 = 곱셈 (바깥 n × 안쪽 n = n²)
5. 입력을 매번 절반으로 = log n (이진탐색·균형트리)
6. 분할정복 = 마스터 정리 (예: 병합정렬 T(n)=2T(n/2)+O(n) = O(n log n))
```

### 예제 (손으로 따라가기)
```python
def f(arr):              # arr 길이 n
    total = 0            # O(1)
    for x in arr:        # n번 반복
        total += x       #   O(1)  → 합쳐서 O(n)
    for i in arr:        # n번
        for j in arr:    #   n번  → 중첩 = O(n²)
            print(i, j)  #     O(1)
    return total         # O(1)
# 합계: O(1)+O(n)+O(n²)+O(1) → 지배항 O(n²)
```

## 6. 흔한 함정

- **O(n²)를 못 알아본다**: 리스트 안에서 `if x in list`(선형검색)를 루프 돌면 숨은 O(n²).
  → Set/dict(O(1) 조회)로 바꾸면 O(n).
- **빅오만 보고 안심**: 빅오가 같아도 상수·캐시가 실제 속도를 가른다.
  → [메모리 계층](../cs-fundamentals/memory-hierarchy-and-latency.md)과 함께 봐야.
- **최선/평균/최악 혼동**: 퀵정렬은 평균 O(n log n)이나 최악 O(n²). 해시는 평균 O(1)이나 최악 O(n).
- **작은 n에선 빅오가 무의미**: n이 작으면 O(n²)가 O(n log n)보다 빠를 수도 (상수 차이).

## 7. 공간 복잡도

시간뿐 아니라 **추가로 쓰는 메모리**도 같은 표기로 본다.
예: 입력 외에 배열 하나 더 만들면 O(n) 공간. 재귀는 **호출 스택 깊이**가 공간 비용.
→ 시간 vs 공간도 트레이드오프 ([트레이드오프 읽는 법](../tradeoffs/reading-tradeoffs-and-metrics.md)).

**Q1.** 다음 각각의 시간복잡도는?
```python
# (a)
for i in range(n):
    for j in range(n):
        ...
# (b)
i = n
while i > 1:
    i = i // 2
# (c)
for i in range(n):
    for j in range(i):   # ← 안쪽이 i까지!
        ...
```

**Q2.** `arr`에 중복이 있는지 검사하는 두 방법의 빅오를 각각 구하고, 왜 다른지 설명하라.
```python
# 방법 A: 이중 루프로 모든 쌍 비교
# 방법 B: set에 넣으며 이미 있으면 중복
```

**Q3.** (측정) Q2의 두 방법을 직접 구현해, n = 1천 / 1만 / 10만에서 시간을 재라.
빅오 예측대로 격차가 벌어지는가? 작은 n에서는 어떤가?
