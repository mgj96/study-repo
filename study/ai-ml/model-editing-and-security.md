# 모델에 심기 — 파인튜닝·모델 편집·백도어 (같은 도구, 방향만 반대)

> 관점: "레이어만 빼서 튜닝되나? 안전을 심나, 거짓·환각을 심나?"에 대한 정리. **같은 메커니즘이 안전도 악의도 심을 수 있다**(대칭). 방어 관점의 교육용.
> 관련: [learning-signal.md](learning-signal.md)(닻·확증편향) · [deep-layers-and-yolo.md](deep-layers-and-yolo.md)(층별 제어) · [for-java-devs.md](for-java-devs.md).

---

## 0. 한 줄 — 같은 칼, 요리도 흉기도

특정 층·뉴런을 골라 조정하는 기술(파인튜닝·모델 편집)은 **안전 행동을 심는 데도, 거짓·백도어를 심는 데도** 쓰인다. 도구는 하나, **방향만 반대.** 그래서 ML **보안**이 중요하다.

---

## 1. "레이어만 빼서 튜닝" — 된다 (PEFT)

- **동결(freeze) 후 일부만 학습**: 나머지 층 얼리고 특정 층만 → **서브클래스에서 메서드 하나만 오버라이드**.
- **LoRA / 어댑터(PEFT, Parameter-Efficient Fine-Tuning)**: 원본 가중치를 **안 건드리고**, 특정 층에 **작은 학습 모듈(diff)** 을 끼워 그것만 학습.
  - **자바 렌즈**: **AOP/데코레이터로 특정 조인포인트에 패치를 주입**하는 것. 원본 `.jar`은 그대로, **얇은 diff만** 얹는다. 배포·롤백도 쉬움.

```
원본 모델(고정) ──▶ [LoRA 패치를 특정 층에 끼움] ──▶ 살짝 다른 행동
   = 원본 .jar 불변         = 데코레이터/AOP diff
```

---

## 2. 특정 지식만 콕 편집 — Model Editing

연구는 **사실이 저장된 위치(주로 중간 MLP 층)를 찾아** 그 가중치를 편집한다.

- **ROME**(Rank-One Model Editing) · **MEMIT**: "에펠탑은 로마에 있다" 같은 **특정 사실을 콕 집어** 바꾼다. FFN 층을 **key-value 연상 메모리**로 보고 해당 항목을 수정(Meng et al. 2022; Geva et al. 2021).
- **Knowledge Neurons**(Dai et al. 2021): 특정 뉴런이 특정 사실을 담고 있음을 보임 → 콕 집어 켜거나 끔.
- **자바 렌즈**: **HashMap의 특정 key에 새 value를 `put`** 하는 것과 비슷. 단 정확 매칭이 아니라 **흐릿한(soft) 매칭**이라, 하나 고치면 **주변 지식이 딸려 바뀌는 부작용**(ripple)이 있다.

---

## 3. 좋은 쪽 — 안전·정렬 심기

- **Safety fine-tuning**: 유해 요청 거절, 정중한 응답 등을 학습으로 심음.
- **RLHF**(사람 피드백 강화학습): "사람이 선호하는 답"에 보상 → 정렬(alignment). (→ [learning-signal.md](learning-signal.md))
- **가드레일**: 입출력 필터를 덧댐(모델 밖 방어).

---

## 4. 나쁜 쪽 — 거짓·환각·백도어 심기 (보안 위협)

같은 기술이 악용되면:

- **데이터 포이즈닝(data poisoning)**: 학습 데이터에 **오염 샘플**을 섞어 특정 오작동을 유도.
- **백도어(backdoor)**: 특정 **트리거 입력**(예: 희귀 단어·패턴)에서만 나쁘게 행동하게 심음. 평소엔 정상이라 **탐지가 어렵다**.
  - **자바 렌즈**: **공급망 공격(supply-chain)** — 의존성에 몰래 악성 코드를 넣거나, 특정 조건에서만 터지는 **로직 폭탄(logic bomb)**.
- **거짓 사실 주입**: 모델 편집으로 틀린 정보를 정답인 양 심음 → 그럴듯하게 퍼뜨림.

> 이건 [learning-signal.md](learning-signal.md)의 "닻이 흔들리면 확증편향"의 **공격 버전** — 신뢰할 수 없는 출처의 모델·데이터·가중치를 그대로 쓰면 위험.

---

## 5. 방어 (defenses)

- **출처 검증(provenance)**: 신뢰할 수 있는 곳의 **가중치·데이터만** 사용. 해시·서명 확인.
- **평가·레드팀(red-teaming)**: 트리거·이상 행동을 **적극적으로 찾아** 시험.
- **데이터 위생**: 학습 데이터 정제·중복·이상치 제거.
- **행동 모니터링**: 배포 후 이상 출력 추적. 가드레일 병행.
- **자바 렌즈**: 낯선 `.jar`(모델)을 무심코 의존성에 넣지 않는다 = **아무 사전학습 모델이나 프로덕션에 올리지 않기.**

## 면접·복습 3단 Q&A

### Q1. 사전학습 모델을 통째로 재학습 안 하고 조금만 손보려면?
- **① 결론** **PEFT**(LoRA·어댑터) — 원본 고정, 작은 패치만 학습.
- **② 원리** 특정 층에 저차원 diff를 끼워 그것만 업데이트. 비용·메모리 절감.
- **③ 확장** 데코레이터처럼 여러 태스크용 패치를 갈아끼움. 원본 재사용.

### Q2. 모델에 특정 사실을 어떻게 넣거나 고치나?
- **① 결론** **모델 편집**(ROME/MEMIT) — 사실이 든 중간 MLP 층 가중치를 수정.
- **② 원리** FFN을 key-value 메모리로 보고 해당 항목을 갱신.
- **③ 확장** 부작용(ripple)·거짓 주입 위험 → 출처·평가 필수.

### Q3. 왜 낯선 모델을 그냥 쓰면 위험한가?
- **① 결론** **백도어·포이즈닝**이 심겨 있을 수 있다.
- **② 원리** 특정 트리거에서만 오작동 → 평소엔 정상이라 탐지 어려움(로직 폭탄).
- **③ 확장** 공급망 공격과 동형. 출처 검증·레드팀으로 방어.

---

## 한 줄 요약
**레이어만 빼서 튜닝은 된다(PEFT), 특정 지식 편집도 된다(ROME/MEMIT).** 이 기술은 **안전을 심을 수도, 거짓·백도어를 심을 수도** 있어 방향만 반대다. 그래서 **모델·데이터·가중치의 출처 검증과 평가**가 핵심. 자바로 치면 **AOP 패치(LoRA)** 와 **공급망 공격(백도어)** 이 같은 주입 지점을 쓰는 셈.
