# CS 암기 Q&A — 자료구조 · 알고리즘

> 형식: **① 결론·WHAT / ② 원리·HOW / ③ 확장·TRADE-OFF**. 외우기용.
> 개념 심화는 [../concept.md](../concept.md) 참고.

---

### Q1. 배열 vs 연결 리스트?
- **① 결론** 배열은 인덱스 접근이 빠르고, 연결 리스트는 삽입·삭제가 빠르다.
- **② 원리** 배열은 연속 메모리라 인덱스 계산으로 O(1) 접근. 연결 리스트는 노드가 포인터로 이어져 앞·중간 삽입 O(1)이나 조회 O(n).
- **③ 확장** 배열은 캐시 효율↑·크기 고정. 연결 리스트는 크기 유연하나 포인터 메모리·캐시 비효율. 접근 패턴으로 선택.

### Q2. 해시맵(hash map)은 어떻게 동작하나? 충돌 해결은?
- **① 결론** 키를 해시 함수로 인덱스로 바꿔 O(1) 평균 조회.
- **② 원리** 해시값으로 버킷 위치 결정 → 저장·조회. 다른 키가 같은 버킷이면 **충돌**.
- **③ 확장** 충돌은 체이닝(연결 리스트)이나 개방 주소법으로 해결. 충돌이 많으면 최악 O(n) → 부하율 넘으면 리사이즈.

### Q3. 스택 vs 큐?
- **① 결론** 스택은 LIFO(후입선출), 큐는 FIFO(선입선출).
- **② 원리** 스택은 한쪽에서 push/pop. 큐는 뒤로 넣고 앞에서 뺌.
- **③ 확장** 스택: 되돌리기·함수 호출·괄호 검사·DFS. 큐: 작업 대기열·BFS. 둘 다 임의 접근은 불가.

### Q4. 이진탐색트리(BST) vs 해시테이블?
- **① 결론** BST는 정렬·범위 검색에, 해시테이블은 단순 키 조회에 강하다.
- **② 원리** BST는 정렬 유지로 O(log n) 탐색·범위 질의 가능. 해시는 O(1) 평균이나 순서가 없다.
- **③ 확장** "정렬·범위"가 필요하면 트리, "빠른 단건 조회"면 해시. BST는 편향 시 O(n)이라 균형 트리(AVL·Red-Black) 사용.

### Q5. 퀵 vs 병합 vs 힙 정렬?
- **① 결론** 셋 다 평균 O(n log n)이지만 성격이 다르다.
- **② 원리** 퀵: 분할, 평균 빠름·제자리(in-place)지만 최악 O(n²). 병합: 안정 정렬·항상 O(n log n)이나 추가 메모리. 힙: 제자리·최악에도 O(n log n)이나 상수 큼.
- **③ 확장** 일반적으론 퀵(실측 빠름), 안정성 필요하면 병합, 메모리 제약·최악 보장 필요하면 힙.

### Q6. BFS vs DFS?
- **① 결론** BFS는 너비 우선(가까운 곳부터), DFS는 깊이 우선(끝까지 갔다 돌아옴).
- **② 원리** BFS는 큐로 레벨 단위 탐색, DFS는 스택(또는 재귀)으로 한 방향 깊게.
- **③ 확장** BFS는 **최단 경로**(가중치 없을 때)에, DFS는 경로 존재·백트래킹·사이클 탐지에. 메모리·용도로 선택.

### Q7. 시간복잡도(Big-O)란?
- **① 결론** 입력이 커질 때 연산 횟수의 증가율. 최악의 경우 기준.
- **② 원리** 상수·계수를 무시하고 지배적인 항만 봄. `O(1) < O(log n) < O(n) < O(n log n) < O(n²)`.
- **③ 확장** 실제 시간이 아니라 규모를 비교하는 척도. 공간복잡도(메모리)와 trade-off도 함께 본다.

### Q8. 동적계획법(DP) vs 분할정복?
- **① 결론** 둘 다 문제를 쪼개지만, DP는 겹치는 부분문제를 저장해 재사용한다.
- **② 원리** 분할정복은 독립적 하위문제로 나눠 합침(병합정렬). DP는 중복 계산을 메모이제이션·테이블로 캐싱.
- **③ 확장** DP는 겹치는 부분문제(피보나치·최단경로)에서 지수→다항으로 줄임. 대신 메모리를 더 쓴다.

### Q9. 그리디(greedy) 알고리즘이란?
- **① 결론** 매 순간 가장 좋아 보이는 선택을 하는 방법.
- **② 원리** 지역 최적을 반복해 전체 답을 구성. 빠르고 단순.
- **③ 확장** 항상 최적은 아님 — "탐욕 선택이 전체 최적"임이 증명될 때만 유효(예: 거스름돈, 최소신장트리). 아니면 DP 필요.

### Q10. 트리 순회(전위/중위/후위)란?
- **① 결론** 트리의 모든 노드를 방문하는 순서. 루트를 언제 방문하냐로 구분.
- **② 원리** 전위(루트→좌→우), 중위(좌→루트→우), 후위(좌→우→루트).
- **③ 확장** 중위 순회는 BST에서 **정렬된 순서**로 나옴. 후위는 삭제·계산에, 전위는 복제·직렬화에 쓰인다.
