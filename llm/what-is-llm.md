# LLM이란 무엇인가

## 핵심 개념

LLM(Large Language Model)은 대규모 텍스트 데이터로 학습된 신경망 모델이다.  
핵심은 **"다음 토큰을 예측"** 하는 것 — 이것만 반복하면 문장이 완성된다.

```
입력: "파이썬으로 리스트를"
예측: "정렬하는" → "방법은" → "sort()" → ...
```

## 작동 원리

```
텍스트 → 토크나이저 → 토큰 ID → Transformer → 확률 분포 → 다음 토큰
```

- **토큰**: 단어보다 작은 단위. "unhappy" → ["un", "happy"] (약 4글자 = 1토큰)
- **Transformer**: 토큰 간 관계(attention)를 계산하는 구조
- **Context Window**: 한 번에 볼 수 있는 토큰 수 — 이게 곧 "기억력"

## 학습 단계

1. **Pretraining** — 인터넷 텍스트 수조 개 토큰으로 다음 토큰 예측 학습
2. **Instruction Tuning** — "질문-답변" 형식으로 미세조정
3. **RLHF** — 사람 피드백으로 좋은 응답 강화

## 개발자가 알아야 할 것

| 개념 | 설명 | 실무 의미 |
|------|------|----------|
| Temperature | 출력 다양성 (0=결정론적, 1=창의적) | 코드생성은 낮게, 브레인스토밍은 높게 |
| Context Window | 한 번에 처리하는 토큰 수 | 큰 코드베이스 분석 가능 여부 결정 |
| Hallucination | 사실처럼 보이는 거짓 출력 | API 명세, 라이브러리 버전은 반드시 검증 |
| Knowledge Cutoff | 학습 데이터 기준일 | 최신 프레임워크는 공식 문서 직접 제공 필요 |

## LLM이 코드를 이해하는 방식

LLM은 AST(추상 구문 트리)를 파싱하지 않는다.  
**토큰 패턴**으로 코드를 인식한다.

- `def`, `class`, `import` 같은 키워드 패턴
- 들여쓰기, 괄호 구조
- 변수명의 의미론적 단서 (`userAge` → 나이 관련)

→ 그래서 **변수명과 함수명이 명확할수록 LLM 분석 품질이 올라간다.**

## 한계

- 코드 실행 불가 (추론만 함) — 실제 버그는 돌려봐야 앎
- 긴 파일은 context 초과 시 앞부분을 "잊음"
- 학습 데이터에 없는 내부 라이브러리/비공개 API는 모름
